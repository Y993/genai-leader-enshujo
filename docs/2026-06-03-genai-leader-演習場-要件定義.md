## 1. 背景・目的

Google Cloud 認定資格の **日本語・無料〜低価格・反復演習特化** の領域は明確に手薄であり、特に GCP 専業の日本語問題集サイトはほぼ存在しない。本プロジェクトはこのギャップを、**新設で競合が手薄な「Generative AI Leader」** から攻める。

- 収益はお小遣い規模（月数千円〜数万円）。本業と両立できる **運用・保守の手間の最小化** が最優先。
- 法人営業・B2B・有料サブスク・課金基盤は **採用しない**（見えない労働を避ける）。
- マネタイズは **アフィリエイト主軸 ＋ AdSense 補助**（いずれも集客が育つフェーズ2以降）。

### 差別化ポイント
1. **正確な日本語解説**（Udemy ですら解説誤りが指摘される領域。正確さが唯一かつ最大の武器）
2. **「読む」でなく「やる」体験**（AI 検索に代替されにくい能動的演習）
3. **GCP 特化**（他サービスは AWS のついでに GCP を足している構造）

## 6. データモデル（問題スキーマ）

問題は **JSON（Astro content collection）** として presentation と分離管理する。試験改定時はデータ差し替えのみで対応。

### 1問あたりの主なフィールド
| フィールド | 型 | 説明 |
|------|------|------|
| `id` | string | 一意なID |
| `category` | enum | `fundamentals` / `google-cloud-offerings` / `improve-output` / `business-strategy` |
| `subTopic` | string | 分野内の小テーマ |
| `difficulty` | enum | `easy` / `medium` / `hard` |
| `tags` | string[] | 横断ミックス・模試・絞り込み用の自由タグ |
| `question` | string | 問題本文 |
| `options` | string[] | 選択肢 |
| `isMultiple` | boolean | 複数選択問題か |
| `correct` | number \| number[] | 正解インデックス（複数選択時は配列） |
| `explanation` | string | 解説（なぜ正解か／なぜ他が誤りか まで） |
| `reference` | string(URL) | 公式ドキュメント等の出典 |
| `relatedMaterials` | object[] | フェーズ2のアフィリエイト枠。フェーズ0は空配列 |

> **将来拡張の担保**: `category` ＋ `tags` ＋ `difficulty` を最初から保持することで、横断ミックス・模試・難易度出し分けを**再設計なし**で追加できる。

## 7. 体験・振る舞い（UX要件）

- 1問表示 → 選択 → **即時に正誤＋解説を展開** → 次の問題へ。
- カテゴリ単位の進捗（解答数・正答率）を **localStorage** に保存し、再訪で復元。
- 結果画面で **誤答だけ再挑戦** できる（フェーズ0の復習導線は軽量）。
- **モバイルファースト**。キーボード操作・コントラスト等のアクセシビリティに配慮。
- 解説末尾・終了画面に **「関連教材」枠**（フェーズ0は非表示またはプレースホルダ）。

## 8. SEO要件

- 各カテゴリページに **オリジナル日本語の導入解説**（検索意図充足＋重複コンテンツ回避）。
- `title` / `meta description` テンプレート、OGP、canonical。
- 構造化データ（Quiz / FAQ schema を適切な範囲で）。
- 内部リンク設計：トップ ⇄ カテゴリ ⇄ 関連カテゴリ（**横断の布石**）。
- `sitemap.xml` 自動生成、`robots.txt`。
- ターゲットKW（ロングテール）例:
  - 「Generative AI Leader 問題集」
  - 「生成AIリーダー 模擬試験 無料」
  - 「GenAI Leader 過去問 日本語」
  - 「Generative AI Leader 試験 対策」
- 計測: **Google Search Console（必須）** ＋ 軽量アクセス解析（GA4 等／実装フェーズで確定）。

## 9. 技術構成

- **Astro**（静的生成・ゼロJSデフォルト。クイズ部分のみ island としてJS付与）。
- 問題データ = **JSON**（content collection）で presentation と分離。
- ホスティング: **GitHub Pages（`Y993` アカウント）**、初期は `https://Y993.github.io/<repo>/`。
- CI/CD: **GitHub Actions で build & deploy を自動化**。
- 運用フロー: **JSONに問題を追記 → push → Actions が再ビルド＆公開**。owner作業は実質「JSON編集」のみ（GitHub Web UI からの編集も可）。
- スタイリング: `frontend-design` スキルで作成（CLAUDE.md準拠）。トーンは**現代的なAI/クラウド系**を想定（和風路線とは別。実装フェーズで確定）。

フェーズ0の実装範囲は **実質 ¥0（完全無料）** で運用できる。

| 項目 | 費用 | 補足 |
|------|------|------|
| Astro / Node.js / 開発ツール | ¥0 | オープンソース |
| GitHubアカウント（Y993） | ¥0 | 既存の無料アカウント |
| GitHub Pages（ホスティング・HTTPS含む） | ¥0 | **public リポジトリなら無料** |
| GitHub Actions（自動build&deploy） | ¥0 | **public リポジトリは実行時間無制限で無料** |
| Search Console / アクセス解析(GA4等) | ¥0 | 無料 |

- **無料維持の条件: リポジトリは `public` 一択**（SEO公開が目的のため問題なし。private は Pages/Actions に制限・課金が絡む）。

- フェーズ0では発生しない費用: 独自ドメイン（年¥1,000〜1,500・フェーズ2以降の任意）、AdSense/アフィリエイト（登録無料）。

## 11. リスク・留意点

| リスク | 対応 |
|------|------|
| 公式問題の転載は著作権侵害 | **公式ガイドの分野・概念に基づくオリジナル作問**。正確さ＝差別化の核 |
| 公式試験の改定で問題が陳腐化 | データ分離設計により差し替え容易（設計済み） |
| AdSense審査は独自ドメイン/十分な分量が有利 | フェーズ2で独自ドメイン化を検討 |
| 新設資格で情報・競合が手薄 | **早期参入の優位**を活かす |
| 解説の誤り＝信頼失墜 | 出典(`reference`)を必ず付与し検証可能にする |
