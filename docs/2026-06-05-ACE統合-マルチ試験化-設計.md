# Google Cloud 認定 演習場 — マルチ試験化（ACE 追加）設計書

> **作成: 2026-06-05** ／ ステータス: 設計合意済み（実装・作問は次セッション）
> **目的:** 既存の単一試験サイト（GenAI Leader）を「試験(exam)」を上位概念とするマルチ試験構造へリファクタし、2つ目の試験 **ACE（Associate Cloud Engineer）** を同一サイトに追加できる土台を作る。ACE の作問は本設計に基づき次セッションでサブエージェント方式で行う。

---

## 0. 背景と現状

- 公開URL: `https://Y993.github.io/genai-leader-enshujo/`（リポジトリ `Y993/genai-leader-enshujo`、public、GitHub Pages）。
- 現状は **単一試験前提**:
  - `src/lib/schema.ts`：`category` が GenAI Leader の4分野の固定 `enum`。
  - `src/data/questions/<category>.json`（4ファイル）＋ `index.ts` が `ALL_QUESTIONS` に集約。
  - `src/data/categories.ts`：4分野メタ（weightPercent 合計100）。
  - ルーティング：`/`（カテゴリ一覧トップ）、`/categories/[slug]/`（クイズ）、法務4ページ。
  - クイズUI：`Quiz.astro` ＋ `global.css`（試験非依存）。
- 現在の問題数: 80問（fund24 / gco28 / imp16 / biz12）。

## 1. 確定した方針（合意済み）

1. **統合方式:** 同一リポジトリ・同一サイトに2試験を統合（別リポジトリにしない）。
2. **URL:** 両試験を**対称プレフィックス**化。旧URLはメタリフレッシュ＋canonicalで退避。
3. **ブランド:** サイト表示名を「GenAI Leader 演習場」→ **「Google Cloud 認定 演習場」** に拡張。リポジトリ名・`base`（`/genai-leader-enshujo/`）は**変更しない**（URL資産維持）。
4. **スコープ（今回）:** 本設計書まで。実装計画・リファクタ・ACE作問は次セッション。

---

## 2. 全体アーキテクチャ：`exam → category → question`

```
EXAMS (exams.ts)
 ├─ genai-leader   … 既存。カテゴリ4 / 問題80
 │    └─ categories: fundamentals, google-cloud-offerings, improve-output, business-strategy
 └─ ace            … 新規。カテゴリ5 / 問題（次セッションで作問）
      └─ categories: setup-environment, plan-configure, deploy-implement, operate, access-security
```

各 `exam` は自身の `categories`（分野メタ）と `questions`（JSON群）を持つ。クイズの解答・採点・進捗ロジックは試験非依存で共通。

### 3.2 `schema.ts` の変更

- `CATEGORY_SLUGS` の固定 `enum` を**廃止**し、`category` は `z.string().min(1)` に緩める。
- カテゴリの妥当性（その試験に属する正しい分野slugか）は **`exams.ts` の定義 ＋ `validateQuestions` の引数**で検証する。
- `Question` 型に **`examId`** を持たせる。ただし JSON には書かず、**読み込み時にディレクトリから注入**する（データ重複・記入ミス防止）。

```ts
// 改修イメージ
export const QuestionSchema = z.object({
  id: z.string().min(1),
  category: z.string().min(1),          // 固定enum廃止
  subTopic: z.string().min(1),
  difficulty: z.enum(['easy','medium','hard']),
  tags: z.array(z.string()).default([]),
  question: z.string().min(1),
  options: z.array(z.string().min(1)).min(2),
  isMultiple: z.boolean(),
  correct: z.union([z.number().int(), z.array(z.number().int()).min(1)]),
  explanation: z.string().min(1),
  reference: z.url(),
  relatedMaterials: z.array(z.object({ label: z.string(), url: z.url() })).default([]),
}).superRefine(/* correct範囲・isMultiple整合は現状維持 */);

export type Question = z.infer<typeof QuestionSchema> & { examId: string };

// examId と「その試験の正しいカテゴリ集合」を受け取り検証
export function validateQuestions(
  raw: unknown[],
  examId: string,
  expectedCategory: string,
  validCategories: readonly string[],
): Question[] { /* category一致＋validCategories所属＋examId注入 */ }
```

### 3.3 `exams.ts`（新規・正典）

```ts
export interface ExamCategory {
  slug: string;
  titleJa: string;
  titleEn: string;
  weightPercent: number;   // 各試験内で合計100
  intro: string;           // SEO用本文
}
export interface Exam {
  id: string;              // 'genai-leader' | 'ace'
  slug: string;            // URLプレフィックス（idと同じでよい）
  titleJa: string;         // 例: 'Generative AI Leader'
  shortJa: string;         // 例: 'GenAI Leader'
  tagline: string;         // トップカード用の一言
  description: string;     // 試験トップのSEO説明
  categories: ExamCategory[];
}
export const EXAMS: Exam[] = [ genaiLeader, ace ];
export function getExam(slug: string): Exam | undefined { ... }
export function getCategory(examSlug: string, catSlug: string): ExamCategory | undefined { ... }
```

- `genaiLeader.categories` は既存 `categories.ts` の4件をそのまま移植。
- `categories.ts` は `exams.ts` に統合して**廃止**する（参照箇所はすべて `exams.ts` の `getCategory(examSlug, catSlug)` に置換）。

// 各試験のJSONを読み、examId・validCategoriesを与えて検証・集約
export const QUESTIONS_BY_EXAM: Record<string, Question[]> = { 'genai-leader': [...], 'ace': [...] };
export const ALL_QUESTIONS: Question[] = Object.values(QUESTIONS_BY_EXAM).flat();

## 5. ACE カテゴリ定義（公式試験ガイド準拠）

| slug | 日本語名 | weightPercent（公式目安） |
|------|----------|---------------------------|
| `setup-environment` | クラウド環境のセットアップ | 17.5 |
| `plan-configure` | クラウドソリューションの計画と構成 | 17.5 |

> **問題数は次セッションで決定**（暫定案：比率に沿って計40問程度から開始、`deploy-implement` を最多に）。idプレフィックス案：`ace-setup` / `ace-plan` / `ace-deploy` / `ace-ops` / `ace-sec`（または単純連番 `ace-001…`）。最終決定は作問前に行う。

## 6. 進捗（localStorage）の試験分離

- 現状キー：`genai-leader-enshujo:progress:v1`（全問混在）。
- 変更：**試験ごとに分離**するため examId をキーに含める：`genai-leader-enshujo:progress:<examId>:v1`。
- `storage.ts` の `loadProgress/saveProgress` を `examId` 引数付きに。`Quiz.astro`・各試験トップの進捗表示も `examId` を渡す。
- 旧キーの移行は不要（GenAI Leader の進捗はローカル端末依存・再取得可能なため、後方互換は最小限でよい。必要なら旧キー→新キーの一度きりコピーを `storage.ts` に入れる選択肢を残す）。

## 7. ブランド／共通レイアウト

- `BaseLayout.astro`：`siteName` を「Google Cloud 認定 演習場」に。ヘッダーのロゴ文言、フッターの免責（「Google / Google Cloud とは無関係の非公式…」）は維持。
- ヘッダーのナビ「カテゴリ」は試験選択トップ（`/`）への導線に調整。試験内ページでは「← 試験トップ」「← 試験一覧」の導線を用意。
- OGP・`<title>`：試験名を含める（例：`{カテゴリ名}の問題集 | {試験名} | Google Cloud 認定 演習場`）。
- クイズUI（`Quiz.astro`・`global.css`）は**変更不要**（試験非依存）。

## 8. 移行手順（既存を壊さない順序・実装時の指針）

1. `exams.ts` を新規作成（genai-leader のカテゴリを移植、ace のカテゴリ定義を追加）。
2. `questions/genai-leader/` を作り既存4 JSON を移動。`index.ts` を exam 集約に改修。
3. `schema.ts` を examId 対応に改修（固定enum廃止）。`validateQuestions` シグネチャ変更。
4. `*.test.ts`（schema/categories/index）を新構造に更新。`npm test` 緑を確認。
5. ルーティング新設：`pages/[exam]/index.astro`、`pages/[exam]/categories/[slug].astro`。
6. `pages/index.astro` を試験選択トップに改修。旧 `pages/categories/[slug].astro` をリダイレクト化。
7. `BaseLayout` のブランド・ナビ・OGP 調整。`storage.ts` を examId 対応に。
8. `npm run build`（Zod全問検証＋全ページ生成）と `npm run preview` で全導線を目視。
9. ここまでで **ACE はカテゴリ枠だけ（問題0）でも成立**するよう設計（空配列でビルドが通ること）。
10. 次に ACE 作問（サブエージェント並列、`docs/2026-06-05-問題追加-作問ガイド.md` の手順を ACE 用に展開）。

> 各カテゴリの問題が0件でもページが破綻しない（「準備中」表示）ようにしておくと、リファクタと作問を分離してデプロイできる。

## 9. テスト方針

- `exams.test.ts`：各試験で `weightPercent` 合計100、slug一意、`intro` 非空、カテゴリ数（genai-leader=4 / ace=5）。
- `schema.test.ts`：examId注入・category妥当性（validCategories外を弾く）・correct整合（現状維持）。
- `index.test.ts`：exam別件数・id全体一意・category が所属試験の集合に含まれること。
- ビルド時 Zod 検証は全試験横断で継続。
