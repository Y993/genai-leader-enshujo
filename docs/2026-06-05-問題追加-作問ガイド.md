## 5. データスキーマと記入規則

ファイル: `src/data/questions/<category>.json`（配列に追記）。1問の形:

```json
{
  "id": "fund-013",
  "category": "fundamentals",
  "subTopic": "コンテキストウィンドウ",
  "difficulty": "medium",
  "tags": ["context-window", "llm"],
  "question": "問題文（日本語・1文〜2文）",
  "options": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  "isMultiple": false,
  "correct": 0,
  "explanation": "正解の理由＋各誤答がなぜ誤りかを日本語で。",
  "reference": "https://cloud.google.com/...",
  "relatedMaterials": []
}
```

- `isMultiple:false` のとき `correct` は**数値**（0始まりindex）。`true` のとき `correct` は**配列**（例 `[0,2]`、最低1要素）。
- `correct` の index は `options` の範囲内（範囲外は Zod が停止）。
- `options` は最低2、各要素は非空文字列。
- `relatedMaterials` は当面 `[]`（フェーズ2でアフィリエイト）。

スキーマ正典: `src/lib/schema.ts`（`QuestionSchema` / `validateQuestions`）。

## 6. reference URL の検証手順（soft-404 に注意）

CLAUDE.md ルール: **HTTP 200 でも中身がエラーの soft-404 がある。本文確認まで行う。**

1. サブエージェントは各 `reference` を **WebFetch で取得**し、本文が当該トピックを説明しているか確認する。
2. リンク切れ・無関係・soft-404 の場合は、`cloud.google.com` 配下の**実在する正しいページ**に差し替える。
3. 推奨ドメイン: `cloud.google.com/`（製品ドキュメント・learn 記事）、`cloud.google.com/vertex-ai/...`、`developers.google.com/`、`ai.google.dev/`（Gemini API）。
4. 不確実な場合は、確実に存在する上位ページ（製品トップ等）にフォールバックし、その旨をメインに報告する。
